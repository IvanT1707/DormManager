## Виявлений процес

### Первинне поселення

- Студент подає заяву на поселення до деканату; конкретне місце надається
  адміністрацією згідно з рішенням і наявними місцями.
- Для оформлення необхідні документ, що посвідчує особу, квитанція про оплату
  проживання, медична довідка, фотографії та інші визначені документи.
- Після оплати й оформлення документів студент проходить інструктаж і отримує
  перепустку; лише після цього має право проживати в гуртожитку.

### Продовження проживання

- Студенти, які вже проживають у гуртожитку і бажають залишитися, подають заяву
  на наступний навчальний рік.
- Рейтингове перепоселення виконується на підставі рейтингу, заяв і
  відсутності заборгованості за проживання.

### Переселення

- Переселення між кімнатами або гуртожитками відбувається за рішенням
  адміністрації Студмістечка за наявності потреби.
- Умови проживання студента при такому переселенні не повинні погіршуватися.

### Виселення

- Виселення виконується на підставі оформленого рішення або звернення: після
  закінчення терміну проживання, за власним бажанням чи за порушення правил.
- Факт припинення проживання має бути відображений у реєстрі мешканців.

## Аудит початкової реалізації

| Вимога процесу | Стан до виправлення | Ризик |
| --- | --- | --- |
| Студент подає заяву на місце, а не обирає кімнату | Студент вибирав "бажану кімнату" для поселення | Хибне враження бронювання місця |
| Продовження проживання | Окремого типу заяви не було | Некоректний сценарій для поточних мешканців |
| Переселення приймається адміністрацією | Студент міг обирати нову кімнату | Невідповідність процедурі |
| Перевірка документів, оплати та оформлення | У заяві були лише статус і опис | Поселення можна було завершити без перевірок |
| Рішення змінює реєстр проживання | Заявка й поселення оброблялися окремо | Статус заявки не гарантував стану реєстру |

## Реалізована відповідність

- Для студента заява `settlement` більше не приймає вибір кімнати.
- Додано тип `renewal` для продовження поточного проживання.
- `relocation` ініціюється комендантом або адміністратором для активного
  мешканця; студент не може подати її самостійно.
- Комендант призначає кімнату в полі `assignedRoomId` під час обробки
  поселення або переселення.
- Для завершення поселення контролюються право на поселення, документи,
  оплата, медична довідка, інструктаж і видача перепустки.
- Для продовження контролюються право на продовження та відсутність
  заборгованості.
- Для переселення підтверджується, що умови не погіршуються.
- Для виселення перед завершенням фіксується підстава.
- Статус `completed` транзакційно створює активне поселення, переносить його
  до нової кімнати або архівує при виселенні.
- Пряме ручне редагування реєстру проживань через API залишено лише
  адміністратору для імпорту або виправлення даних; комендант працює через
  заяви та рішення.

## Межі автоматизації

DormManager зберігає результат адміністративної перевірки, але ще не
обчислює академічний рейтинг студентів: вихідні рейтингові дані належать
інститутам і деканатам. Наступним розширенням може бути імпорт рейтингового
списку та автоматичне ранжування заяв на продовження проживання.
